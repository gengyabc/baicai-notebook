# Query Confidence Rules

When answering knowledge questions from this vault:

For folder semantics and duplicate control, see `.opencode/rules/core-vault.md`. For promotion boundaries, see `.opencode/rules/promotion-policy.md`.

1. Read root `index.md` first to understand vault structure and layer purposes.
2. Read `workbook/wiki/index.md` to locate relevant wiki pages for the query.
3. Prefer the relevant `workbook/wiki/` pages for stable answers.
4. Fall back to `workbook/resources/` for evidence and provenance.
5. Use `workbook/brainstorm/` only as tentative synthesis.
6. Use `workbook/my-work/` only for current intent, context, and active priorities.

## Answer style

- State the confidence level when the answer is not fully grounded.
- Separate confirmed facts from working hypotheses.
- Cite the folder or note source that supports the answer when practical.
- If evidence conflicts, say so plainly instead of collapsing the conflict.
